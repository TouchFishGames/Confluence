TouchFishGames : 常规使用方法

Created by 杨志庄, last modified on 12月 16, 2022

  • NetworkBehaviour

Mirror本身继承MonoBehaviour实现了网络同步版的NetworkBehaviour。涉及网络通信的脚本使用NetworkBehaviour（如玩家移动、保存实体的生命值等属性），仅本地运算的脚本使用传统的MonoBehaviour（如UI）。

  • NetworkIdentity

所有需要通过Mirror来进行同步的脚本都依赖于网络唯一标识符。换言之，使用了Mirror的脚本所挂载的GameObject必须同时挂载NetworkIdentity脚本。

※挂载NetworkBehaviour脚本时会自动追加NetworkIdentity，但是需要在Inspector窗口手动修改NetworkIdentity的设置。

  • SyncVar

变量特性。与[SerlizeField]同样，追加在变量前使用。使用后该变量会自动同步。

[SyncVar] [SerializeField] private float hp;

  • Command

函数特性。RPC。此类函数一律以Cmd开头命名。由客户端发起在服务端执行。有权限验证。

       [Command]
		public void CmdPlayerTakeDamage(float damage)
		{
			hp -= damage;
			hp = Mathf.Min(hp, maxHp);
			if (hp > Mathf.Epsilon)
				return;
			hp = 0f;
			CmdPlayerDie();
		}

		[Command]
		private void CmdPlayerDie()
		{
			Debug.Log($"{GetComponent<NetworkIdentity>().netId} died.");
		}
